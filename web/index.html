Hva er JusJob?

JusJob henter automatisk data fra norske rettskilder, bygger en søkeindeks og publiserer den via GitHub Pages. En Zotero-plugin lar deg søke i alle kildene samlet og importere treff direkte til Zotero-biblioteket ditt.

Indeks-skjema

Felles feltformat for alle kilder i search-index.json. Se full dokumentasjon for per-kilde-detaljer og lenkestrategi.

{ "id": "rettspraksis-12345", // unik ID: {kilde}-{nøkkel} "source": "rettspraksis.no", // kildenavn "type": "rettsavgjørelse", // dokumenttype "title": "HR-2020-123-A", // tittel "court_or_body": "Høyesterett", // instans / organ / departement "url": "https://...", // lenke til dokument "snippet": "Første 300 tegn ...", // tekstutdrag / sammendrag // Planlagte felt (under implementasjon): "date": "2020-03-15", // dato (avgjørelse / ikrafttredelse / publisering) "eli": "/eli/lov/1902/05/22", // europeisk lovidenfikator (kun lover/forskrifter) "citation": "HR-2020-123-A", // offisiell sitering (Rt., HR-, RG-, NOU, Prop.) "keywords": ["erstatning", "..."], // stikkord "pdf_url": "https://...", // direktelenke til PDF "authors": "Dommer Bergsjø", // forfatter / dommere "links": { "eli": "/eli/lov/2001/06/15-59", // lov dette dokumentet gjelder "hjemmel": ["LOV-2001-06-15-59-§7"], // hjemmelsreferanser (forskrifter) "relaterte": ["stortinget-12345"] // relaterte saker i indeksen } }

Lenkestrategi

Dokumenter lenkes via felles eli-identifikatorer: Lov ↔ Forskrift (via hjemmel) ↔ Stortingssak (via tittelmatching) ↔ Innstilling/Lovvedtak (via tilknyttede saker) ↔ Rettsavgjørelse (via lovhenvisninger i tekst). Implementeres trinnvis i build_index.py etter at alle kilder er på plass.
